# 预警弹窗场景过滤修复说明

## 问题描述

在多场景应用中，不同场景（药品、库存、食品）的预警弹窗会在所有场景中提示，导致用户在药品场景下也能看到库存或食品场景的预警通知。

## 问题原因

后端通知服务在生成和获取待推送通知时，没有根据用户当前的使用场景（`current_mode`）进行过滤，导致所有场景的预警都会被推送。

## 修复方案

### 1. 后端修改

修改文件：`backend/app/services/notification.py`

#### 修改点 1：生成通知时过滤场景

在 `generate_notifications_for_user` 方法中：

```python
async def generate_notifications_for_user(self, user_id: int) -> int:
    """为用户生成待推送通知（从未读预警中）"""
    # 获取用户当前场景
    user_result = await self.db.execute(
        select(User).where(User.id == user_id)
    )
    user = user_result.scalar_one_or_none()
    if not user:
        return 0
    
    current_mode = user.current_mode or "medicine"
    
    # 获取所有未读且未被忽略的预警，并根据用户当前场景过滤
    result = await self.db.execute(
        select(Warning)
        .where(
            Warning.is_read == False,
            Warning.is_dismissed == False,
            Warning.mode == current_mode  # 只获取当前场景的预警
        )
        .options(selectinload(Warning.medicine))
        .order_by(Warning.created_at.desc())
    )
    # ...
```

#### 修改点 2：获取待推送通知时过滤场景

在 `get_pending_notifications` 方法中：

```python
async def get_pending_notifications(self, user_id: int, limit: int = 10) -> list[dict]:
    """获取待推送的通知（分别推送，每次返回一条）"""
    # 获取用户当前场景
    user_result = await self.db.execute(
        select(User).where(User.id == user_id)
    )
    user = user_result.scalar_one_or_none()
    if not user:
        return []
    
    current_mode = user.current_mode or "medicine"
    
    # 获取未推送且未读的通知，并过滤场景
    result = await self.db.execute(
        select(Notification)
        .outerjoin(Warning, Notification.warning_id == Warning.id)
        .where(
            Notification.user_id == user_id,
            Notification.is_pushed == False,
            Notification.is_read == False,
            Notification.push_date == date.today(),
            (Notification.warning_id == None) | (Warning.is_dismissed == False),
            # 只返回当前场景的通知
            (Notification.warning_id == None) | (Warning.mode == current_mode)
        )
        .order_by(Notification.created_at.asc())
        .limit(min(limit, remaining))
    )
    # ...
```

## 修复效果

修复后：

1. ✅ 用户在**药品场景**下，只会收到药品相关的预警弹窗
2. ✅ 用户在**库存场景**下，只会收到库存相关的预警弹窗
3. ✅ 用户在**食品场景**下，只会收到食品相关的预警弹窗
4. ✅ 切换场景后，预警弹窗会自动切换到对应场景的内容

## 技术细节

### 数据库字段

- `users.current_mode`：用户当前使用的场景（medicine/inventory/food）
- `warnings.mode`：预警所属的场景（medicine/inventory/food）

### 过滤逻辑

通过 SQL JOIN 和 WHERE 条件，确保：
1. 生成通知时，只从当前场景的预警中生成
2. 获取待推送通知时，只返回当前场景的通知
3. 用户切换场景后，自动切换到新场景的预警

## 注意事项

- 修改后需要重启后端服务才能生效
- 已经生成的历史通知不会被删除，但不会再弹出
- 用户切换场景后，新的预警会根据新场景生成
