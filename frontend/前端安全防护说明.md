# 前端安全防护说明

## 安全防护措施概览

本系统实现了多层次的前端安全防护，防止通过浏览器开发者工具查看关键信息和攻击。

## 1. 开发者工具检测

### 检测方法

| 方法 | 原理 |
|-----|------|
| 窗口大小检测 | 开发者工具打开时窗口内外尺寸差异 |
| Debugger时间检测 | debugger语句在开发者工具打开时会暂停 |
| Console检测 | 通过console.log触发getter检测 |
| Element检测 | 通过console.dir触发getter检测 |

### 检测到后的处理

- 清除本地存储的敏感数据（token等）
- 显示安全警告页面
- 锁定页面功能

## 2. 控制台保护

- 重写所有console方法（log, debug, info, warn, error等）
- 防止通过iframe恢复console
- 在控制台显示安全警告信息

## 3. 快捷键和右键菜单禁用

禁用的快捷键：
- F12 - 开发者工具
- Ctrl+Shift+I - 开发者工具
- Ctrl+Shift+J - 控制台
- Ctrl+Shift+C - 元素选择器
- Ctrl+U - 查看源代码
- Ctrl+S - 保存页面

禁用右键菜单和拖拽。

## 4. 反调试保护

- 持续执行debugger检测
- 检测断点设置
- 检测关键函数是否被篡改（fetch, XMLHttpRequest等）

## 5. 数据保护

### Token加密存储

Token在localStorage中以混淆形式存储：
```javascript
// 存储时
value = btoa(value.split('').reverse().join(''))

// 读取时
value = atob(value).split('').reverse().join('')
```

### 全局对象冻结

生产环境下冻结关键原型对象：
- Object.prototype
- Array.prototype
- Function.prototype

## 6. 网络请求保护

- 验证所有请求URL是否合法
- 自动添加安全请求头
- 拦截跨域请求并记录日志

## 7. 页面完整性检查

- 使用MutationObserver监控DOM变化
- 检测并移除可疑的脚本注入
- 移除HTML注释

## 8. 复制保护

- 检测复制内容是否包含敏感信息
- 敏感内容复制时自动脱敏

## 9. 会话保护

- 30分钟无操作自动登出
- 页面切换时检查会话状态
- 监听用户活动（鼠标、键盘、滚动等）

## 10. CSP内容安全策略

在index.html中配置：
```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self';
  script-src 'self' 'unsafe-inline' 'unsafe-eval';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: blob:;
  font-src 'self' data:;
  connect-src 'self' http://localhost:* http://192.168.*:*;
" />
```

## 11. 其他安全Meta标签

```html
<meta http-equiv="X-Content-Type-Options" content="nosniff" />
<meta http-equiv="X-Frame-Options" content="DENY" />
<meta http-equiv="X-XSS-Protection" content="1; mode=block" />
<meta name="referrer" content="strict-origin-when-cross-origin" />
<meta name="robots" content="noindex, nofollow, noarchive, nosnippet" />
```

## 工具函数

### 敏感数据脱敏

```typescript
import { maskSensitiveData } from '@/utils/security'

// 手机号：138****1234
maskSensitiveData('13812341234', 'phone')

// 邮箱：a***@example.com
maskSensitiveData('admin@example.com', 'email')

// 身份证：110***********1234
maskSensitiveData('110101199001011234', 'idcard')

// 姓名：张*
maskSensitiveData('张三', 'name')
```

### XSS过滤

```typescript
import { escapeHtml } from '@/utils/security'

// 过滤HTML特殊字符
escapeHtml('<script>alert(1)</script>')
// 输出: &lt;script&gt;alert(1)&lt;/script&gt;
```

### 安全JSON解析

```typescript
import { safeJSONParse } from '@/utils/security'

// 安全解析JSON，失败时返回默认值
safeJSONParse('invalid json', { default: true })
```

| `src/main.ts` | 入口文件，初始化安全模块 |
| `index.html` | HTML安全配置 |

## 注意事项

1. 所有安全防护仅在生产环境（`import.meta.env.PROD`）启用
2. 开发环境下不会禁用开发者工具
3. 安全防护不能100%阻止攻击，但可以大大增加攻击难度
4. 建议配合后端安全措施一起使用
