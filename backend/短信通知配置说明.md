# 短信通知功能配置说明

## 功能概述

短信通知功能用于向用户发送临期药品预警短信，支持以下类型：
- **临期预警**：药品即将过期时发送提醒
- **过期预警**：药品已过期时发送提醒
- **低库存预警**：库存不足时发送提醒

## 阿里云短信服务配置

### 1. 开通阿里云短信服务

1. 登录 [阿里云控制台](https://console.aliyun.com/)
2. 搜索"短信服务"并开通
3. 进入 [短信服务控制台](https://dysms.console.aliyun.com/)

### 2. 创建 AccessKey

1. 进入 [AccessKey 管理](https://ram.console.aliyun.com/manage/ak)
2. 创建 AccessKey（建议使用子账号，仅授予短信权限）
3. 保存 AccessKey ID 和 AccessKey Secret

### 3. 申请短信签名

1. 在短信服务控制台，进入"国内消息" -> "签名管理"
2. 点击"添加签名"
3. 签名名称建议使用：`药品管理系统` 或您的公司/应用名称
4. 等待审核通过（通常1-2个工作日）

### 4. 申请短信模板

在"模板管理"中创建以下模板：

#### 临期预警模板（必需）
```
模板名称：临期药品预警
模板内容：您好，药品${name}（批次${batch}）将在${days}天后过期，请及时处理。
变量说明：
- name: 药品名称
- batch: 批次号
- days: 剩余天数
```

#### 过期预警模板（可选）
```
模板名称：过期药品预警
模板内容：您好，药品${name}已过期${days}天，请立即处理。
变量说明：
- name: 药品名称
- days: 过期天数
```

#### 低库存预警模板（可选）
```
模板名称：低库存预警
模板内容：您好，药品${name}库存不足，当前库存${stock}件，请及时补货。
变量说明：
- name: 药品名称
- stock: 当前库存
```

### 5. 配置环境变量

在 `backend/.env` 文件中添加以下配置：

```env
# 阿里云短信配置
ALIYUN_SMS_ACCESS_KEY_ID=你的AccessKeyId
ALIYUN_SMS_ACCESS_KEY_SECRET=你的AccessKeySecret
ALIYUN_SMS_SIGN_NAME=药品管理系统
ALIYUN_SMS_TEMPLATE_CODE=SMS_123456789

# 可选：单独的过期和低库存模板
ALIYUN_SMS_EXPIRED_TEMPLATE_CODE=SMS_987654321
ALIYUN_SMS_LOW_STOCK_TEMPLATE_CODE=SMS_456789123

# 启用短信服务
SMS_ENABLED=true
SMS_DAILY_LIMIT=10
```

### 6. 创建数据库表

运行以下命令创建短信相关数据表：

```bash
cd backend
python create_sms_tables.py
```

### 7. 重启后端服务

```bash
python run.py
```

## 使用说明

### PC端配置

1. 登录管理后台
2. 进入"系统管理" -> "短信通知"
3. 开启短信通知开关
4. 选择需要接收的通知类型
5. 设置通知手机号（可选，默认使用账号绑定的手机号）
6. 设置每日通知时间

### API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/sms/status` | GET | 获取短信服务状态 |
| `/api/sms/config` | GET | 获取用户短信配置 |
| `/api/sms/config` | PUT | 更新短信配置 |
| `/api/sms/send` | POST | 发送单条预警短信 |
| `/api/sms/send-batch` | POST | 批量发送预警短信 |
| `/api/sms/test` | POST | 发送测试短信（仅管理员） |
| `/api/sms/logs` | GET | 获取发送记录 |
| `/api/sms/stats` | GET | 获取发送统计 |

## 费用说明

阿里云短信服务按条计费：
- 国内短信：约 0.045 元/条
- 每日限额：默认 10 条/用户（可在配置中调整）

## 注意事项

1. **签名审核**：首次使用需要等待签名审核通过
2. **模板审核**：模板内容需要符合阿里云规范
3. **发送限制**：每个用户每天有发送上限，避免骚扰
4. **手机号格式**：仅支持中国大陆手机号（11位）
5. **测试短信**：建议先使用测试功能验证配置是否正确

## 常见问题

### Q: 短信发送失败，提示"签名不存在"
A: 检查签名是否审核通过，签名名称是否与配置一致

### Q: 短信发送失败，提示"模板不存在"
A: 检查模板是否审核通过，模板CODE是否正确

### Q: 短信发送失败，提示"权限不足"
A: 检查 AccessKey 是否有短信服务权限

### Q: 如何查看发送记录？
A: 在 PC 端"系统管理" -> "短信通知"页面可以查看发送记录
