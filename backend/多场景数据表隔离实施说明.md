# 多场景数据表隔离实施说明

## 当前状态
数据库中已存在以下表：
- `medicine_categories` - 药品分类表
- `medicine_items` - 药品物品表  
- `medicine_batches` - 药品批次表
- `medicines` - 旧的药品表（可能存在）
- `batches` - 旧的批次表（可能存在）

## 实施步骤

### 1. 添加 current_mode 字段
已在 User 模型中添加 `current_mode` 字段，默认值为 'medicine'。

需要手动执行SQL：
```sql
ALTER TABLE users ADD COLUMN current_mode VARCHAR(20) DEFAULT 'medicine' NOT NULL COMMENT '当前使用场景';
```

### 2. 创建其他场景的数据表
需要为以下场景创建数据表：
- inventory (库存管理)
- food (食品管理)
- equipment (设备管理)
- document (文档管理)
- custom (自定义场景)

每个场景需要3个表：
- `{mode}_categories` - 分类表
- `{mode}_items` - 物品表
- `{mode}_batches` - 批次表

### 3. 手动执行SQL脚本
由于Alembic迁移遇到表已存在的问题，建议手动执行以下SQL：

```sql
-- 1. 添加 current_mode 字段（如果不存在）
ALTER TABLE users ADD COLUMN IF NOT EXISTS current_mode VARCHAR(20) DEFAULT 'medicine' NOT NULL COMMENT '当前使用场景';

-- 2. 为 inventory 场景创建表
CREATE TABLE IF NOT EXISTS inventory_categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    description VARCHAR(200),
    sort_order INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS inventory_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    code VARCHAR(50) NOT NULL UNIQUE,
    barcode VARCHAR(50),
    category_id INT NOT NULL,
    specification VARCHAR(100),
    unit VARCHAR(20) NOT NULL,
    manufacturer VARCHAR(100),
    price FLOAT,
    description TEXT,
    is_deleted BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES inventory_categories(id),
    INDEX idx_name (name),
    INDEX idx_code (code),
    INDEX idx_barcode (barcode)
);

CREATE TABLE IF NOT EXISTS inventory_batches (
    id INT AUTO_INCREMENT PRIMARY KEY,
    item_id INT NOT NULL,
    batch_number VARCHAR(50),
    quantity INT DEFAULT 0,
    production_date DATE,
    expiry_date DATE NOT NULL,
    inbound_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    zero_stock_date DATETIME,
    remark VARCHAR(200),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (item_id) REFERENCES inventory_items(id),
    INDEX idx_item_id (item_id),
    INDEX idx_batch_number (batch_number)
);

-- 插入默认分类

-- 3-6. 重复以上步骤为 food, equipment, document, custom 场景创建表
-- （省略，结构相同，只需替换表名前缀）
```
