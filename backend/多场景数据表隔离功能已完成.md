## 功能特性

### 1. 数据完全隔离
不同场景使用完全独立的数据表，数据互不影响：
- 药品管理场景只能访问 `medicine_*` 表
- 库存管理场景只能访问 `inventory_*` 表
- 其他场景同理

### 2. 动态表名
后端使用动态模型创建技术，根据用户的 `current_mode` 自动选择对应的数据表：
```python
# 用户当前场景：inventory
service = ModeService(db, user.current_mode)
ItemModel = service.ItemModel  # 自动映射到 inventory_items
```

### 3. 前后端联动
前端切换场景时自动调用后端API更新用户配置：
```typescript
// 切换到库存管理
await authApi.updateUserMode('inventory')
uni.setStorageSync('app_mode', 'inventory')
uni.reLaunch({ url: '/pages/index/index' })
```

### 4. 术语动态替换
前端根据场景自动替换界面术语：
- 药品管理：药品、盒、效期、批号
- 库存管理：商品、件、保质期、批次
- 设备管理：设备、台、维护期、编号

## 使用方式

### 用户操作流程
1. **登录系统** - 获取当前场景（默认：药品管理）
2. **选择场景** - 在场景选择页面选择使用场景
3. **确认切换** - 系统自动更新用户配置并重新加载
4. **使用功能** - 所有数据操作自动使用对应场景的数据表

### 开发者使用
```python
# 后端服务层
from app.services.mode_service import ModeService

class MyService(ModeService):
    def __init__(self, db, user_mode):
        super().__init__(db, user_mode)
    
    async def get_items(self):
        # 自动使用对应场景的表
        result = await self.db.execute(
            select(self.ItemModel)
        )
        return result.scalars().all()
```

## 技术亮点

### 1. 零停机部署
使用 `CREATE TABLE IF NOT EXISTS` 确保脚本可以重复执行，不会影响现有数据。

### 2. 智能错误处理
Python脚本自动识别"表已存在"等警告，不会中断执行。

### 3. 模型缓存
动态创建的模型会被缓存，避免重复创建，提高性能。

### 4. 类型安全
使用SQLAlchemy的类型注解，确保代码的类型安全。
