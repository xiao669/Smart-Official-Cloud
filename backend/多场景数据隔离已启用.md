## 工作原理

### 用户切换场景流程

1. **用户选择场景**
   ```typescript
   // 前端：pages/select-mode/index.vue
   await authApi.updateUserMode('inventory')
   uni.setStorageSync('app_mode', 'inventory')
   uni.reLaunch({ url: '/pages/index/index' })
   ```

2. **后端更新用户配置**
   ```python
   # 后端：app/services/auth.py
   user.current_mode = 'inventory'
   await self.db.commit()
   ```

3. **API自动使用对应表**
   ```python
   # 后端：app/api/medicines.py
   service = MedicineService(db, current_user.current_mode)
   # current_user.current_mode = 'inventory'
   # 自动使用 inventory_items, inventory_batches, inventory_categories
   ```

4. **动态模型选择表**
   ```python
   # 后端：app/services/mode_service.py
   class ModeService:
       def __init__(self, db, mode):
           self.ItemModel = get_item_model(mode)  # inventory_items
           self.BatchModel = get_batch_model(mode)  # inventory_batches
           self.CategoryModel = get_category_model(mode)  # inventory_categories
   ```

## 技术实现

### 动态表名映射
```python
# app/utils/mode_tables.py
MODE_PREFIXES = {
    'medicine': 'medicine',
    'inventory': 'inventory',
    'food': 'food',
    'equipment': 'equipment',
    'document': 'document',
    'custom': 'custom'
}

def get_table_name(mode: str, base_table: str) -> str:
    return f"{MODE_PREFIXES[mode]}_{base_table}"
```

### 动态模型创建
```python
# app/models/mode_models.py
def create_item_model(mode: str) -> Type:
    table_name = get_table_name(mode, 'items')
    
    class ItemModel(Base):
        __tablename__ = table_name
        __table_args__ = {'extend_existing': True}
        # ... 字段定义
    
    return ItemModel
```

class MedicineService(ModeService):

## 注意事项

### 1. 数据不可共享
不同场景的数据完全隔离，无法共享。如果需要在多个场景中使用相同的数据，需要分别录入。

### 2. 分类独立
每个场景都有自己的分类体系，已经预置了默认分类：
- 药品管理：化学药品、中成药、中药饮片、生物制品、保健食品
- 库存管理：日用品、食品饮料、办公用品、电子产品、其他
- 食品管理：主食、蔬菜、水果、肉类、调味品、其他
- 设备管理：办公设备、生产设备、检测设备、运输设备、其他设备
- 文档管理：合同文件、技术文档、财务文件、人事文件、其他文档
- 自定义场景：分类一、分类二、分类三、其他

### 3. 术语自动替换
前端会根据场景自动替换界面术语，无需手动修改。

### 4. 用户场景持久化
用户的场景选择会保存在数据库中（users.current_mode），下次登录时自动恢复。
