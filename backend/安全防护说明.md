# 医药管理系统 - 安全防护说明

## 安全防护措施概览

本系统实现了多层次的安全防护措施，确保系统在生产环境中的安全性。

### 1. 速率限制（Rate Limiting）

防止暴力破解和DDoS攻击。

| 接口类型 | 限制 | 时间窗口 |
|---------|------|---------|
| 全局请求 | 100次 | 1分钟 |
| 登录接口 | 5次 | 1分钟 |
| 敏感接口 | 20次 | 1分钟 |

超过限制返回 `429 Too Many Requests`。

### 2. 登录失败锁定

防止暴力破解密码。

- 最大失败次数：5次（5分钟内）
- 锁定时间：15分钟
- 锁定后返回剩余解锁时间

### 3. IP黑名单

自动封禁恶意IP。

- 可疑行为阈值：10次
- 临时封禁时间：1小时
- 支持永久黑名单

### 4. 请求验证

防止常见Web攻击。

- SQL注入检测
- XSS攻击检测
- 路径遍历检测

检测到攻击返回 `400 Bad Request`。

### 5. 安全响应头

所有响应自动添加安全头：

```
X-Frame-Options: DENY                    # 防止点击劫持
X-Content-Type-Options: nosniff          # 防止MIME嗅探
X-XSS-Protection: 1; mode=block          # XSS保护
Content-Security-Policy: default-src 'self'  # 内容安全策略
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: geolocation=(), microphone=(), camera=()
```

### 6. 密码安全

- 使用 bcrypt 算法哈希密码
- 密码强度要求：
  - 最少8位
  - 必须包含大写字母
  - 必须包含小写字母
  - 必须包含数字
  - 必须包含特殊字符
- 禁止常见弱密码

### 7. JWT Token 安全

- 自动生成安全密钥
- Token有效期：8小时
- 包含签发时间和类型验证
- 强制要求过期时间

### 8. 审计日志

记录所有敏感操作到 `audit.log`：

- 登录成功/失败
- 敏感操作（用户管理、系统配置等）
- 安全事件（攻击检测、IP封禁等）

## 生产环境配置

### 1. 环境变量配置

在 `.env` 文件中配置：

```env
# 关闭调试模式
DEBUG=false

# 设置安全的密钥（必须修改！）
SECRET_KEY=your-very-long-and-secure-secret-key-here

# 配置允许的域名
CORS_ORIGINS=["https://your-domain.com"]
ALLOWED_HOSTS=["your-domain.com", "www.your-domain.com"]
```

### 2. HTTPS 配置

生产环境必须启用HTTPS。在 `security_middleware.py` 中取消注释：

```python
response.headers["Strict-Transport-Security"] = "max-age=31536000; includeSubDomains"
```

### 3. 数据库安全

- 使用强密码
- 限制数据库访问IP
- 定期备份数据

### 4. 服务器安全

- 使用防火墙
- 定期更新系统
- 监控异常流量

## 安全事件响应

### 检测到攻击时

1. 系统自动记录到审计日志
2. 累计可疑行为达到阈值自动封禁IP
3. 管理员可查看日志分析攻击来源

### 账户被锁定时

1. 用户需等待锁定时间结束
2. 管理员可手动解锁（通过数据库）
3. 建议用户修改密码

## 反爬虫系统

### 1. 设备指纹识别

基于多维度特征生成唯一设备指纹：
- User-Agent
- Accept-Language
- Accept-Encoding
- 屏幕分辨率（前端传递）
- 时区信息

检测同一指纹来自多个IP的可疑行为，支持指纹黑名单。

### 2. 行为分析引擎

实时分析访问行为，检测异常模式：

| 检测项 | 阈值 | 分数 |
|-------|------|------|
| 高频请求 | >60次/5分钟 | +20 |
| 请求间隔规律 | 方差<0.1 | +25 |
| 顺序访问模式 | >90%顺序 | +15 |
| 高错误率 | >30% | +20 |
| 响应处理过快 | <50ms | +10 |
| 频繁访问敏感接口 | >10次 | +15 |

### 3. User-Agent分析

检测已知爬虫特征：
- python-requests, curl, wget
- Scrapy, BeautifulSoup
- Selenium, Puppeteer, Playwright
- 各种Bot和Spider

验证是否为合法浏览器（Chrome, Firefox, Safari, Edge）。

### 4. 蜜罐陷阱

设置隐藏路径捕获爬虫：
- `/api/admin/secret`
- `/api/internal/data`
- `/.env`
- `/wp-admin`
- `/phpinfo.php`

触发蜜罐返回假数据迷惑爬虫，多次触发自动封禁。

### 5. 爬虫评分卡

综合评估是否为爬虫：

| 维度 | 权重 |
|-----|------|
| UA可疑 | 25分 |
| 行为异常 | 最高30分 |
| 指纹可疑 | 15分 |
| 触发蜜罐 | 40分 |
| 无Cookie | 10分 |
| 无Referer | 5分 |
| 请求头异常 | 15分 |

**封禁阈值：60分**

### 6. 响应头

每个响应返回 `X-Crawler-Score` 头，显示当前请求的爬虫评分。
