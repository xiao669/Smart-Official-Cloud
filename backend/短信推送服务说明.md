# 短信推送服务详细说明

## 一、服务概述

短信推送服务用于向用户发送药品预警通知短信，包括：
- **临期预警**：药品即将过期提醒
- **过期预警**：药品已过期提醒
- **低库存预警**：药品库存不足提醒

### 技术架构
- **短信服务商**：阿里云短信服务
- **后端框架**：FastAPI + SQLAlchemy
- **数据库**：MySQL

## 四、发送流程

### 4.0 自动推送流程（定时任务）

系统启动后会自动运行定时任务，每小时检查一次：

```
系统启动 → 启动定时任务
    ↓
每小时检查当前时间
    ↓
查询 notify_time = 当前小时 且 sms_enabled = true 的用户
    ↓
为每个用户调用 send_batch_warnings_sms()
    ↓
发送未读且未忽略的预警短信
```

**配置说明：**
- 用户可在短信配置中设置 `notify_time`（如 09:00）
- 系统会在该时间点自动发送预警短信
- 默认通知时间为每天早上 9:00

### 4.1 单条预警短信发送流程

```
用户点击发送 → send_warning_sms()
    ↓
检查预警是否被忽略 (is_dismissed)
    ↓ 未忽略
检查全局开关 (sms_enabled)
    ↓ 已开启
检查API配置 (access_key_id, access_key_secret)
    ↓ 已配置
检查用户配置 (user.sms_enabled)
    ↓ 已开启
检查每日限额 (daily_limit)
    ↓ 未超限
检查该类型短信开关 (expiry/expired/low_stock_sms_enabled)
    ↓ 已开启
检查是否已发送过 (sms_logs)
    ↓ 未发送
创建发送记录 → 调用阿里云API → 更新记录状态
```

### 4.2 批量发送流程

```
定时任务/手动触发 → send_batch_warnings_sms()
    ↓
检查发送条件
    ↓
查询未读且未忽略的预警 (is_read=False, is_dismissed=False)
    ↓
循环发送（检查每日限额）
    ↓
返回发送统计
```

## 五、费用控制机制

### 5.1 多重防护措施

| 防护层级 | 机制 | 说明 |
|---------|------|-----|
| 全局开关 | sms_enabled | 可一键关闭所有短信发送 |
| 每日限额 | daily_limit | 限制每用户每天最多发送条数（默认10条） |
| 用户开关 | user.sms_enabled | 用户可自行关闭短信通知 |
| 类型开关 | expiry/expired/low_stock_sms_enabled | 可分别控制各类型短信 |
| 去重机制 | sms_logs | 同一预警只发送一次 |
| 忽略过滤 | is_dismissed | 被忽略的预警不发送短信 |

### 5.2 费用估算

假设配置：
- 每日限额：10条/用户
- 阿里云短信单价：约 0.045 元/条

| 场景 | 每日费用 | 每月费用 |
|-----|---------|---------|
| 1个用户，满额发送 | 0.45元 | 13.5元 |
| 5个用户，满额发送 | 2.25元 | 67.5元 |
| 10个用户，满额发送 | 4.5元 | 135元 |

**建议**：根据实际需求调整 daily_limit，避免不必要的费用支出。

## 七、阿里云配置指南

### 7.1 开通短信服务
1. 登录阿里云控制台
2. 搜索"短信服务"并开通
3. 完成企业实名认证（个人认证有限制）

### 7.2 创建AccessKey
1. 进入 RAM 访问控制
2. 创建用户，勾选"编程访问"
3. 为用户添加 `AliyunDysmsFullAccess` 权限
4. 保存 AccessKey ID 和 Secret

### 7.3 申请短信签名
1. 进入短信服务控制台
2. 国内消息 → 签名管理 → 添加签名
3. 签名类型选择"验证码/通知短信"
4. 等待审核通过（1-2个工作日）

### 7.4 申请短信模板
需要申请3个模板：

**临期预警模板**
```
模板内容：您的药品${name}（批次${batch}）将在${days}天后过期，请及时处理。
变量说明：name-药品名称，batch-批次号，days-剩余天数
```

**过期预警模板**
```
模板内容：您的药品${name}已过期${days}天，请立即处理！
变量说明：name-药品名称，days-过期天数
```

**低库存预警模板**
```
模板内容：您的药品${name}库存不足，当前库存${stock}件，请及时补货。
变量说明：name-药品名称，stock-当前库存
```

## 八、常见问题

### Q1: 短信发送失败怎么办？
**排查步骤：**
1. 检查 sms_logs 表中的 error_message
2. 常见错误：
   - `isv.BUSINESS_LIMIT_CONTROL`：触发流控，稍后重试
   - `isv.MOBILE_NUMBER_ILLEGAL`：手机号格式错误
   - `isv.TEMPLATE_MISSING_PARAMETERS`：模板参数不完整
   - `isv.AMOUNT_NOT_ENOUGH`：账户余额不足

### Q2: 为什么同一预警不会重复发送？
系统会检查 sms_logs 表，如果该预警已有成功发送记录，则不会重复发送。

### Q3: 被忽略的预警会发送短信吗？
不会。系统在发送前会检查 `warning.is_dismissed`，被忽略的预警会跳过。

### Q4: 如何完全关闭短信功能？
方法1：在系统配置中将 `sms_enabled` 设为 false
方法2：清空 `access_key_id` 和 `access_key_secret`

### Q5: 每日限额用完后怎么办？
等待第二天自动重置，或由管理员调整 `daily_limit` 配置。

## 九、安全注意事项

1. **AccessKey 保护**：不要在前端暴露 AccessKey Secret
2. **手机号脱敏**：日志中显示的手机号会自动脱敏（如 138****8000）
3. **频率限制**：阿里云有默认频率限制，同一手机号1分钟内最多1条
4. **费用监控**：建议在阿里云设置费用预警，避免异常消费
