# 药品管理系统 - 数据库自动备份使用说明

## 功能特性

✅ **多级备份策略**
- 每日备份：保留7天
- 每周备份：保留4周
- 每月备份：保留12个月

✅ **自动化功能**
- 自动压缩（gzip）
- 自动验证备份完整性
- 自动清理过期备份
- 自动计算MD5校验和

✅ **可选功能**
- 邮件通知（备份成功/失败）
- 阿里云OSS云存储同步
- 备份统计报告

## 快速开始

### 1. 配置环境

复制配置文件并修改：

```bash
cd backend/scripts
cp backup_config.env .env
```

编辑 `.env` 文件，修改数据库连接信息：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的数据库密码
DB_NAME=medicine_admin
```

### 2. 手动执行备份

```bash
# 进入脚本目录
cd backend/scripts

# 执行每日备份
python backup_database.py backup --type daily

# 执行每周备份
python backup_database.py backup --type weekly

# 执行每月备份
python backup_database.py backup --type monthly
```

### 3. 配置自动备份

**Windows系统：**
```bash
# 双击运行
setup_backup_windows.bat
```

**Linux系统：**
```bash
chmod +x setup_backup_cron.sh
./setup_backup_cron.sh
```

## 配置邮件通知（可选）

编辑 `.env` 文件：

```env
# QQ邮箱示例
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USER=your_qq@qq.com
SMTP_PASSWORD=你的授权码（不是QQ密码）
NOTIFY_EMAIL=接收通知的邮箱
```

**获取QQ邮箱授权码：**
1. 登录QQ邮箱
2. 设置 → 账户 → POP3/SMTP服务 → 开启
3. 生成授权码

## 配置云存储（可选）

支持阿里云OSS，编辑 `.env` 文件：

```env
OSS_ENABLED=true
OSS_ACCESS_KEY=你的AccessKey
OSS_SECRET_KEY=你的SecretKey
OSS_BUCKET=你的Bucket名称
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
```

需要安装oss2库：
```bash
pip install oss2
```

## 定时任务说明

### Windows任务计划

| 任务名称 | 执行时间 | 备份类型 |
|---------|---------|---------|
| MedicineAdmin_DailyBackup | 每天 02:00 | 每日备份 |
| MedicineAdmin_WeeklyBackup | 每周日 03:00 | 每周备份 |
| MedicineAdmin_MonthlyBackup | 每月1号 04:00 | 每月备份 |

查看任务：
```bash
schtasks /query /tn "MedicineAdmin_DailyBackup"
```

### Linux Crontab

```cron
# 每天凌晨2点执行每日备份
0 2 * * * python /path/to/backup_database.py backup --type daily

# 每周日凌晨3点执行每周备份
0 3 * * 0 python /path/to/backup_database.py backup --type weekly

# 每月1号凌晨4点执行每月备份
0 4 1 * * python /path/to/backup_database.py backup --type monthly
```

## 常见问题

### Q: 备份失败提示 "mysqldump: command not found"
A: 需要安装MySQL客户端工具，或将mysqldump添加到系统PATH

### Q: 备份文件太大怎么办？
A: 
1. 减少保留天数（修改配置）
2. 定期清理历史数据
3. 使用云存储

### Q: 如何验证备份是否可用？
A: 
1. 查看备份日志确认无错误
2. 检查 .json 元数据文件中的校验和
3. 在测试环境恢复验证

### Q: 邮件通知收不到？
A: 
1. 检查SMTP配置是否正确
2. QQ邮箱需要使用授权码，不是登录密码
3. 检查垃圾邮件文件夹

---

## 最佳实践

1. **定期检查备份日志** - 确保备份正常执行
2. **定期测试恢复** - 每月至少测试一次恢复流程
3. **异地备份** - 启用云存储，防止本地灾难
4. **监控磁盘空间** - 确保有足够空间存储备份
5. **保护备份文件** - 设置适当的文件权限
