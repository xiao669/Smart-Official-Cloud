# 云服务器部署安全配置指南

本指南介绍如何在云服务器上安全部署药品管理系统，使用IP直接访问。

## 一、配置HTTPS（自签名证书）

### 1.1 生成自签名SSL证书

在云服务器上执行以下命令：

```bash
# 创建证书目录
mkdir -p /etc/nginx/ssl
cd /etc/nginx/ssl

# 生成私钥
openssl genrsa -out server.key 2048

# 生成证书签名请求（CSR）
# 注意：Common Name 填写你的服务器IP，如 123.45.67.89
openssl req -new -key server.key -out server.csr

# 生成自签名证书（有效期365天）
openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt

# 设置权限
chmod 600 server.key
chmod 644 server.crt
```

### 1.2 配置Nginx启用HTTPS

创建Nginx配置文件 `/etc/nginx/conf.d/medicine-admin.conf`：

```nginx
# HTTP重定向到HTTPS
server {
    listen 80;
    server_name _;  # 匹配所有请求
    return 301 https://$host$request_uri;
}

# HTTPS配置
server {
    listen 443 ssl;
    server_name _;

    # SSL证书配置
    ssl_certificate /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    
    # SSL安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 前端静态文件
    location / {
        root /var/www/medicine-admin/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 1.3 重启Nginx

```bash
# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

### 1.4 浏览器信任自签名证书

由于是自签名证书，浏览器会提示"不安全"，需要手动信任：

**Chrome浏览器：**
1. 访问 `https://你的IP`
2. 点击"高级" → "继续前往（不安全）"
3. 或者导出证书添加到系统信任列表

**移动端App：**
- Android: 需要在设置中安装证书
- iOS: 需要在设置 → 通用 → 关于 → 证书信任设置中启用

## 二、修改JWT密钥

### 2.1 生成强密钥

在服务器上生成一个安全的随机密钥：

```bash
# 方法1：使用Python生成
python3 -c "import secrets; print(secrets.token_hex(32))"

# 方法2：使用openssl生成
openssl rand -hex 32

# 输出示例：a1b2c3d4e5f6...（64位十六进制字符串）
```

### 2.2 修改后端配置

编辑 `backend/.env` 文件：

```env
# 数据库配置
DATABASE_URL=mysql+aiomysql://root:你的数据库密码@localhost/medicine_admin

# JWT配置 - 必须修改为你生成的密钥！
JWT_SECRET_KEY=你生成的64位密钥
JWT_ALGORITHM=HS256
JWT_EXPIRE_MINUTES=1440

# OCR配置（如果使用）
OCR_API_KEY=你的OCR密钥
OCR_SECRET_KEY=你的OCR密钥

# DeepSeek AI配置（如果使用）
DEEPSEEK_API_KEY=你的DeepSeek密钥
```

### 2.3 验证配置生效

重启后端服务后，旧的Token会自动失效，用户需要重新登录。

```bash
# 重启后端服务
cd /path/to/backend
python run.py
# 或使用systemd服务
systemctl restart medicine-backend
```

## 三、定期备份数据库

### 3.1 创建备份脚本

创建文件 `/opt/scripts/backup_mysql.sh`：

```bash
#!/bin/bash

# 配置
DB_USER="root"
DB_PASS="你的数据库密码"
DB_NAME="medicine_admin"
BACKUP_DIR="/opt/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
KEEP_DAYS=30  # 保留30天的备份

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/${DB_NAME}_${DATE}.sql

# 压缩备份文件
gzip $BACKUP_DIR/${DB_NAME}_${DATE}.sql

# 删除超过30天的旧备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +$KEEP_DAYS -delete

# 记录日志
echo "$(date): 备份完成 - ${DB_NAME}_${DATE}.sql.gz" >> $BACKUP_DIR/backup.log
```

### 3.2 设置脚本权限

```bash
chmod +x /opt/scripts/backup_mysql.sh
```

### 3.3 配置定时任务（每天凌晨2点自动备份）

```bash
# 编辑crontab
crontab -e

# 添加以下行（每天凌晨2点执行）
0 2 * * * /opt/scripts/backup_mysql.sh
```

### 3.4 手动执行备份

```bash
# 立即执行一次备份
/opt/scripts/backup_mysql.sh

# 查看备份文件
ls -la /opt/backups/mysql/
```

### 3.5 恢复数据库（如需要）

```bash
# 解压备份文件
gunzip /opt/backups/mysql/medicine_admin_20251219_020000.sql.gz

# 恢复数据库
mysql -uroot -p medicine_admin < /opt/backups/mysql/medicine_admin_20251219_020000.sql
```

## 五、快速部署命令汇总

```bash
# 1. 安装依赖
yum install -y nginx mysql-server python3.11 openssl

# 2. 生成SSL证书
mkdir -p /etc/nginx/ssl
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/server.key \
  -out /etc/nginx/ssl/server.crt \
  -subj "/CN=你的服务器IP"

# 3. 生成JWT密钥
python3 -c "import secrets; print(secrets.token_hex(32))"

# 4. 创建备份目录和脚本
mkdir -p /opt/backups/mysql /opt/scripts

# 5. 启动服务
systemctl enable nginx mysql
systemctl start nginx mysql
```

## 六、常见问题

### Q: 浏览器提示证书不安全怎么办？
A: 自签名证书会有这个提示，点击"高级"→"继续访问"即可。内部使用不影响安全性。

### Q: 忘记修改JWT密钥会怎样？
A: 系统会自动生成一个随机密钥，但每次重启服务密钥会变化，导致用户需要重新登录。建议固定配置。

### Q: 备份文件太大怎么办？
A: 脚本已自动压缩，如果还是太大，可以：
1. 减少保留天数（修改KEEP_DAYS）
2. 使用增量备份
3. 定期清理历史数据

### Q: 如何迁移到新服务器？
A: 
1. 备份数据库
2. 复制整个项目代码
3. 在新服务器恢复数据库
4. 修改配置文件中的IP地址
